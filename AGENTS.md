# AKNOA AGENTS.md

## 目的

このファイルは、AKNOAプロジェクトでCodexが実装を行う際のガイドラインです。

AKNOAは長期運用を前提とした個人開発プロジェクトです。

実装速度だけでなく、保守性・可読性・一貫性を重視してください。

---

# プロジェクト概要

AKNOAは生活を便利にするWebツールを提供するサービスです。

現在の機能

- サブスク管理

今後追加予定

- 健康管理
- 家計・固定費管理
- FIREシミュレーター
- ライフスタイルツール
- ブログ

---

# 技術スタック

- Next.js（App Router）
- React
- TypeScript
- Tailwind CSS
- Vercel

---

# 開発方針

以下を最優先してください。

- 可読性
- 保守性
- シンプルな設計
- 型安全
- コンポーネントの再利用性

オーバーエンジニアリングは避けてください。

---

# 最重要ルール

実装は依頼された内容のみ行ってください。

勝手に以下を行わないでください。

- リファクタリング
- 命名変更
- ディレクトリ変更
- UI変更
- ライブラリ追加
- 設計変更

必要な場合は実装前に理由を説明してください。

---

# ディレクトリ構成

```
src
│
├── app
│
├── components
│   ├── Header.tsx
│   ├── Footer.tsx
│   └── （アプリ全体で再利用するUI）
│
├── data
│
├── hooks
│   （アプリ全体で利用する共通Hooks）
│
├── features
│   └── feature-name
│       ├── components
│       ├── hooks
│       ├── types
│       ├── utils
│       └── ...
```

---

# ディレクトリルール

## src/components

アプリ全体で再利用するUIのみ配置してください。

例

- Header
- Footer
- Button
- Modal

---

## src/hooks

アプリ全体で利用する共通Hooksのみ配置してください。

例

- useLocalStorage
- useDebounce
- useMediaQuery

---

## src/features

各機能専用のコードを配置してください。

例

```
features/subscriptions
```

その中に

- components
- hooks
- types
- utils

を配置してください。

機能専用コンポーネントはfeatures配下へ配置してください。

---

# TypeScript

- anyは禁止
- 型安全を維持する
- TypeScriptエラーを残さない

---

# React

- Functional Componentを使用する
- Hooksのルールを守る
- Propsは必要最小限にする
- コンポーネントは責務を小さく保つ

---

# コーディングルール

- 関数は短くする
- 命名は分かりやすくする
- 深いネストを避ける
- 可読性を最優先する
- 重複コードを減らす

---

# Tailwind CSS

Tailwind CSSを使用してください。

基本デザイン

背景

- white

アクセントカラー

- cyan-600

カード

- rounded-2xl
- border-zinc-200

インラインスタイルは禁止です。

既存デザインとの統一感を維持してください。

# レスポンシブ

モバイルファーストで実装してください。

以下を考慮してください。

- タブレット

---

# コメント

コメントは必要最小限にしてください。

コメントを書くのは

- 複雑な処理
- 業務ルール
- 意図が分かりにくい処理

のみです。

コード自体を読みやすくしてください。

---

# リファクタリング

依頼がない限り以下は行わないでください。

- ディレクトリ変更
- ファイル移動
- 命名変更
- 共通化
- アーキテクチャ変更

必要な場合は理由を説明してください。

---

# 完了前チェック

必ず以下を確認してください。

- TypeScriptエラーなし
- ESLintエラーなし（設定されている場合）
- 未使用importなし
- 未使用変数なし
- console.logを残さない
- npm run build が成功する
- 既存機能を壊していない

---

# 作業完了時の報告

必ず以下を報告してください。

1. 変更したファイル一覧

2. 修正内容

3. build結果

4. lint結果

5. TypeScriptエラーの有無

6. 影響範囲

7. 依頼内容以外を変更していないこと

build や lint が失敗した場合は、

- 今回の変更が原因なのか
- 既存コードが原因なのか

を明確に区別して報告してください。

---

# 出力ルール

コードだけでなく以下も簡潔に説明してください。

- 変更理由
- 実装内容
- 影響範囲

---

# コードを書く前

実装前に既存コードを十分読み、

- ディレクトリ構成
- 命名規則
- コンポーネント設計
- デザイン
- Tailwindの書き方

を理解した上で実装してください。

新しい書き方を持ち込むより、既存コードとの一貫性を優先してください。

---

# AIの役割

CodexはAKNOAの実装パートナーです。

目的は「新しいコードを書くこと」ではなく、

**既存プロジェクトの品質・一貫性を維持しながら、安全に実装すること**です。

常に既存設計を尊重し、依頼内容を正確に実装してください。
